# 自定义控件开发指南

本文档基于项目实际 API 和多次开发经验总结，融合了官方文档的全部核心内容，并新增避坑要点和最佳实践。读完本文，五分钟即可上手开发自己的控件。

---

## 一、核心概念与前置知识

### 1.1 你只需要做两件事

| 步骤 | 操作 | 说明 |
|------|------|------|
| ① | 在 `src/` 下新建一个 `.ts` 文件，例如 `MyWidget.ts` | 文件名任意 |
| ② | 在 `src/main.ts` 中 `import './MyWidget'` | 加一行导入即可 |

不需要修改任何系统文件。`registerCustomWidget` 从项目已有 API（`src/config/customWidgetAPI.ts`）导出，你只管调用。

### 1.2 生效示例（项目中已有）

打开 `src/examples/customWidgetDemo.ts`，里面注册了两个示例控件：

| 控件 | type | 说明 |
|------|------|------|
| 评分控件 | `starRating` | 显示星星 + 当前值/最大星数 |
| 数据卡片 | `statCard` | 标题 + 大号数值 + 副标题 |

这两个示例在 `src/main.ts` 中通过 `import './examples/customWidgetDemo'` 引入。如果要禁用它们，只需注释掉该 import 行。

### 1.3 注册 ↔ 生效全流程

```
① 创建注册文件（如 src/MyWidget.ts）
   └─ 从 src/config/customWidgetAPI.ts 导入 registerCustomWidget（项目已有）
   └─ 调用 registerCustomWidget({ type, label, icon, jsonConfig, htmlTemplate })

② 在 src/main.ts 中 import 该文件
   └─ import './MyWidget'

③ 刷新页面（Ctrl+F5）
   └─ 左侧库面板「自定义控件」出现新图标

④ 拖入画布
   └─ 控件显示，属性面板自动渲染 defaultProps 中的字段

⑤ 属性面板修改值
   └─ 模板中 {{prop}} 实时更新

⑥ 样式编辑器修改
   └─ mappedProps 中的属性可编辑，设计区+预览同步

⑦ 预览/导出
   └─ HTML 模板渲染 + structuralCSS + mappedProps + styleData.sub 合并输出
```

### 1.4 关键文件说明（你无需修改）

| 文件 | 谁负责 |
|------|--------|
| `src/config/customWidgetAPI.ts` | 系统已提供，**不需要你写** |
| `src/config/widgetRegistry.ts` | 系统已提供，**不需要你写** |
| `src/components/widgets/CustomWidget.vue` | 系统已提供，**不需要你写** |
| `src/main.ts` | **你只需要加一行 import** |
| `src/你的文件.ts` | **你写注册代码** |

## 二、最小可用模板（复制即用）

```ts
import { registerCustomWidget } from '@/config/customWidgetAPI'

registerCustomWidget({
    type: 'myFirstWidget',          // 唯一标识，不能与内置控件重名
    label: '我的控件',               // 左侧库面板显示的名称
    icon: '✅',                     // 图标（emoji 或 Unicode）

    jsonConfig: {
        label: '我的控件',           // 与上面的 label 一致
        icon: '✅',                 // 与上面的 icon 一致

        defaultStyleData: {
            base: {
                width: 200,         // 必填，数字单位 px
                height: 100,        // 必填，否则控制点失效
                backgroundColor: '#f5f5f5',
                borderWidth: 1,
                borderStyle: 'solid',
                borderColor: '#ddd',
                borderRadius: 4,
                // ⚠️ 不要加 transition 属性，否则拖拽会延迟
            },
            sub: {},                // 至少空对象
            pseudo: {},             // 至少空对象
        },

        mappedProps: [],            // 至少空数组
        structuralCSS: {},          // 至少空对象
        computedProps: [],          // 至少空数组

        defaultProps: {
            text: 'Hello',
            visible: true,          // 必填
            disabled: false,        // 必填
        },
    },

    htmlTemplate: '<div style="padding: 10px;">{{text}}</div>',
    cssString: '',
})
```

拖入画布后：你会看到一个灰色方块，里面显示 "Hello"。属性面板会出现一个 "text" 文本框。

## 三、字段详解（逐层剖析）

### 3.1 三层 DOM 结构（必读）

```
┌─ #widget_id（外层容器）──────────────────┐ ← base 管这里
│  background, border, border-radius,       │   （背景、边框、圆角、
│  box-shadow, display, flex...             │    阴影、flex 布局等）
│                                           │
│  ┌─ .custom-widget-content ────────────┐  │
│  │  v-html 渲染 htmlTemplate 的内容     │  │
│  │                                      │  │
│  │  <div class="card-title">    ← sub / mappedProps / structuralCSS
│  │  <div class="card-value">    ← 这些管模板里的 class 样式
│  │                              │  │
│  └──────────────────────────────┘  │
│                                           │
│  :hover → pseudo 管这里                   │
└───────────────────────────────────────────┘
```

- **base** 样式只在外层容器上，**不会穿透**到模板内部的元素。
- `border-radius`、`padding`、`background` 等**不可继承属性**，各层独立设置。
- `color`、`font-family` 等**可继承属性**会从容器传到子元素。

### 3.2 type / label / icon（顶层字段）

| 字段 | 类型 | 说明 |
|------|------|------|
| `type` | `string` | 控件唯一标识，不能与内置控件（button、input 等）重名。拖入画布后控件 DOM id 为 `{type}_1`、`{type}_2`…… |
| `label` | `string` | 左侧库面板显示的名称 |
| `icon` | `string` | 可以是 emoji（`'📊'`）或 Unicode 转义（`'\uD83D\uDCCA'`） |

### 3.3 defaultStyleData.base — 外层容器默认样式

**作用 DOM**：`#widget_id`（控件根元素）。

- 必填：`width` 和 `height`（数字，自动加 `px`）。缺 `height` 会导致控制点失效。
- 可用：`backgroundColor`、`borderRadius`、`boxShadow`、`display`、`flexDirection`、`padding`、`margin`、`border` 等。
- ❌ 绝对不要写 `transition`！会导致拖拽移动时卡顿延迟。
- ❌ 不要在这里写内部元素的样式。

```ts
base: {
    width: 320,
    height: 220,
    backgroundColor: '#ffffff',
    borderColor: '#d9d9d9',
    borderWidth: 1,
    borderStyle: 'solid',
    borderRadius: 12,
    boxShadow: '0 2px 8px rgba(0,0,0,0.06)',
    padding: '16px',
    display: 'flex',
    flexDirection: 'column',
}
```

### 3.4 defaultStyleData.sub — 内部子元素默认样式

**作用 DOM**：`htmlTemplate` 中带有 `class` 的元素。

- 键名必须以 `.` 开头，与模板中的 class 名一致。
- 值：驼峰式 CSS 属性名 → 属性值（字符串不加 px，数字自动加）。
- 用户不可见，仅作为初始值（会被 `mappedProps` 覆盖）。

```ts
sub: {
    '.card-title':   { fontSize: '18px', color: '#333' },
    '.card-value':   { fontSize: '32px', fontWeight: 'bold' },
    '.card-subtitle': { fontSize: '12px', color: '#999' },
}
```

### 3.5 defaultStyleData.pseudo — 外层容器伪类样式

**作用 DOM**：`#widget_id` 的伪类（目前支持 `hover`）。

```ts
pseudo: {
    hover: { boxShadow: '0 8px 20px rgba(0,0,0,0.1)', borderColor: '#409eff' }
}
```

### 3.6 mappedProps — 可编辑的子元素样式

**作用**：用户可以在「样式编辑」标签页中修改这些属性。

- 数组，每个元素包含 `selector`（以 `.` 开头）、`base`（默认样式）、`pseudo`（该子元素的伪类样式）。
- 与 `sub` 的关系：`sub` 提供初始默认值，`mappedProps` 的 `base` 会覆盖 `sub` 中的同名属性。

```ts
mappedProps: [
    {
        selector: '.card-title',
        base: { fontSize: '18px', color: '#1f2d3d' },
        pseudo: { hover: { color: '#ff6b6b' } }
    },
    {
        selector: '.card-value',
        base: { fontSize: '32px', color: '#409eff' },
        pseudo: {}
    }
]
```

### 3.7 structuralCSS — 不可编辑的结构样式

**作用**：强制布局属性（`display`、`position`、`gap`），用户无法修改，始终生效。

- 键名是选择器（以 `.` 开头），值是 CSS 声明字符串数组。
- 典型用途：`display: flex`、`text-align: center`、`margin`、`padding` 等布局属性。

```ts
structuralCSS: {
    '.card-title':   ['display: block', 'text-align: center'],
    '.card-value':   ['display: block', 'text-align: center', 'line-height: 1.2'],
    '.card-footer':  ['display: flex', 'justify-content: space-between', 'align-items: center'],
}
```

### 3.8 computedProps — 动态计算属性

根据控件**实际宽高或属性**自动计算 CSS 值，避免写死数值。每条是一个计算规则：

| 字段 | 说明 |
|------|------|
| `selector` | CSS 选择器（`#id` 或 `.class`） |
| `cssProp` | 要写入的 CSS 属性名（kebab-case） |
| `template` | 模板字符串，用 `{{变量}}` 引用实际值 |

**支持的预置变量**：

| 变量 | 取值来源 | 示例结果 |
|------|----------|------|
| `{{halfH}}` | `widget.style.height / 2` | `18`（高度 36px 时） |
| `{{height}}` | `widget.style.height` | `36` |
| `{{width}}` | `widget.style.width` | `200` |
| `{{progressValue}}` | `widget.progressValue` | `50`（进度值 0-100） |
| `{{headerHeight}}` | `widget.headerHeight` | `32` |
| `{{layout}}` | `widget.layout` | `'vertical'` |
| `{{lineStyle}}` | `widget.style.lineStyle` | `'solid'` |
| `{{borderColor}}` | `widget.style.borderColor` | `'#ddd'` |

**示例**：开关按钮用 `{{halfH}}` 让圆角随高度自适应——

```ts
// toggle.json 中的真实配置
computedProps: [
  {
    selector: '#id',
    cssProp: 'border-radius',
    template: '{{halfH}}px'   // 高度36 → 输出 border-radius: 18px
  }
]
```

如果不需要计算属性，写空数组 `[]` 即可。**注意**：模板变量只支持上面的预置列表，不能自定义表达式。

### 3.9 defaultProps — 业务属性（自动生成属性面板）

每个 key 会成为控件的直接属性（如 `widget.title`）。模板中用 `{{key}}` 引用。

必须包含 `visible: true` 和 `disabled: false`，否则控件可能不显示或序列化异常。

**类型自动识别并渲染表单**：

| 值示例 | 类型 | 属性面板渲染 |
|--------|:--:|------|
| `'hello'` | `string` | 文本输入框 |
| `42` | `number` | 数字输入框 |
| `true` | `boolean` | 复选框 |
| `['折线图','柱状图']` | `array` | 下拉框（默认取首项） |
| `'#1890ff'` | 以 `#` 开头的颜色字符串 | 颜色选择器 |

> 颜色检测用正则 `/^#[0-9a-fA-F]{3,8}$/`，只有标准 hex 格式才触发。`'red'`、`'rgb(255,0,0)'` 不会变成颜色选择器。

```ts
defaultProps: {
    title: '数据看板',
    value: 98,
    showFooter: true,
    theme: ['默认主题', '暗黑主题'],
    accentColor: '#409eff',
    visible: true,
    disabled: false,
}
```

### 3.10 htmlTemplate — HTML 模板

使用 `{{propName}}` 引用 `defaultProps` 中的属性。

- **不支持任何表达式、三元运算、函数调用**。例如 `{{showExtra ? 'A' : 'B'}}` 不会生效，会原样输出。
- 条件渲染请使用 `data-*` 属性配合 `cssString` 实现。
- `class` 名必须与 `structuralCSS`、`mappedProps`、`sub` 中的选择器一致（去掉开头的 `.`）。

```html
<div class="card-wrapper" data-theme="{{theme}}">
    <div class="card-title">{{title}}</div>
    <div class="card-value" style="color: {{accentColor}};">{{value}}</div>
    <div class="card-footer" data-visible="{{showFooter}}">底部内容</div>
</div>
```

### 3.11 cssString — 额外 CSS

字符串形式，会被注入到设计区、预览页、导出文件。

**用途**：实现复杂联动（如复选框显隐、主题切换）、关键帧动画等。

**重要**：CSS 规则须加 `!important` 覆盖 `structuralCSS` 或其他生成样式。

```css
/* 复选框联动：showFooter=false 时隐藏底部 */
.card-footer[data-visible="false"] { display: none !important; }

/* 主题切换：dark 模式下改背景 */
[data-theme="暗黑主题"] .card-wrapper {
    background: #1e1e2f !important;
}
[data-theme="暗黑主题"] .card-title { color: #cdd6f4 !important; }
[data-theme="暗黑主题"] .card-value { color: #89b4fa !important; }

/* 动画 */
.card-wrapper { animation: fadeIn 0.3s ease; }
@keyframes fadeIn {
    from { opacity: 0; transform: translateY(5px); }
    to   { opacity: 1; transform: translateY(0); }
}
```

## 四、数据绑定链路（修改一个值经历了什么）

以修改 `title` 为例：

```
┌─── registerCustomWidget 定义阶段 ─────────────────────────────────────────┐
│                                                                          │
│  defaultProps: {                                                         │
│    title: '总用户数',    ← ① 定义 key = "title"，默认值 = "总用户数"      │
│  }                                                                       │
│                                                                          │
│  htmlTemplate:                                                           │
│    '<div class="card-title">{{title}}</div>'    ← ② 模板中 {{title}}      │
│                                                    引用同一个 key         │
│                                                                          │
├─── 控件创建时（拖入画布） ────────────────────────────────────────────────┤
│                                                                          │
│  widget.title = '总用户数'     ← ③ defaultProps 的值复制到控件对象        │
│                                                                          │
├─── 属性面板渲染 ──────────────────────────────────────────────────────────┤
│                                                                          │
│  遍历 defaultProps，跳过 visible/disabled:                                │
│    key = "title", value = "总用户数" → 渲染为 <input type="text">         │
│                                                                          │
├─── 用户在属性面板修改 title = "新标题" ───────────────────────────────────┤
│                                                                          │
│  store.updateWidget(widget.id, { title: "新标题" })                      │
│    → widget.title = "新标题"           ← ④ 更新控件对象                    │
│                                                                          │
├─── 模板重新渲染（设计区 + 预览） ─────────────────────────────────────────┤
│                                                                          │
│   把 '{{title}}' 替换为 widget.title                                     │
│    → <div class="card-title">新标题</div>  ← ⑤ 实时生效                   │
│                                                                          │
└──────────────────────────────────────────────────────────────────────────┘
```

**核心规则**：`defaultProps` 的 key 和 `htmlTemplate` 的 `{{key}}` **必须同名**。

| 位置 | 写法 | 作用 |
|------|------|------|
| `defaultProps.title` | `title: '总用户数'` | 定义属性默认值 → 创建时变为 `widget.title` |
| `htmlTemplate` | `{{title}}` | 替换为 `widget.title` 的值 |
| 属性面板 | 读取 `widget.title` | 渲染表单字段，用户可修改 |
| 预览导出 | 替换 `{{title}}` 为 `widget.title` | 实时反映修改后的值 |

## 五、四个样式层的关系（一图看懂）

同一选择器的样式合并成一条规则：

```css
#statCard_1 .card-title {
    display: block;         /* ← structuralCSS      */
    text-align: center;     /* ← structuralCSS      */
    color: #999;            /* ← mappedProps 默认值  */
    font-size: 12px;        /* ← mappedProps 默认值  */
}
/* 用户修改后，color/font-size 变为新值 */
/* display/text-align 始终不变（不可编辑）*/
```

| 层 | 来源 | 作用 DOM | 可编辑？ | 说明 |
|------|------|----------|---------|------|
| `defaultStyleData.base` | jsonConfig | `#widget_id` 外层容器 | ❌ | 背景、边框、圆角、阴影、flex |
| `structuralCSS` | jsonConfig | 模板内 `.class` 元素 | ❌ | 布局结构（display, gap, padding） |
| `mappedProps` | jsonConfig | 模板内 `.class` 元素 | ✅ | 视觉属性（color, fontSize） |
| `defaultStyleData.sub` | jsonConfig | 模板内 `.class` 元素 | ❌ | 与 mappedProps 合并输出 |
| 样式编辑器修改 | 用户操作 | — | — | 覆盖 mappedProps 默认值 |

## 七、常见错误与避坑清单

| 错误现象 | 原因 | 解决方案 |
|----------|------|----------|
| 拖拽控件时卡顿、延迟 | `base` 中写了 `transition` | ❌ 删除所有 `transition` |
| 控件高度异常、控制点失效 | `base` 缺少 `height` | ✅ 同时写 `width` 和 `height` |
| 控件不显示或预览消失 | `defaultProps` 缺少 `visible: true` | ✅ 添加 `visible: true` |
| 下拉框/颜色选择器不出现 | 值不是数组或不是 `#` 开头的颜色 | ✅ 数组写 `['A','B']`，颜色写 `'#1890ff'` |
| 模板中 `{{prop}}` 没被替换 | 使用了表达式（`{{a?b:c}}`）或特殊字符 | ✅ 只用纯变量名 `{{propName}}` |
| 复选框联动不生效 | CSS 没加 `!important` 或选择器写错 | ✅ 用 `[data-xxx="false"]` + `!important` |
| 子元素样式无法编辑 | 没在 `mappedProps` 中定义 | ✅ 每个可编辑子元素都要加条目 |
| 样式编辑器中修改后无效 | 该属性在 `structuralCSS` 中重复了 | ✅ 布局放 `structuralCSS`，视觉放 `mappedProps` |
| 数组属性初始值变成逗号拼接 | widget 的数组属性未转换 | ✅ 创建控件时自动取首项 |
| `base` 样式应用到内部元素无效 | 误以为 `base` 会穿透到子元素 | ✅ `base` 只控制外层，内部用 `sub`/`mappedProps`/`structuralCSS` |

## 八、必填字段速查表

| 字段路径 | 要求 | 若缺失后果 |
|----------|:--:|------|
| `type` | 唯一字符串 | 无法注册 |
| `label` | 字符串 | 库面板无名 |
| `jsonConfig.label` | 同 `label` | 可能报错 |
| `jsonConfig.icon` | 同 `icon` | 可能报错 |
| `jsonConfig.defaultStyleData.base.width` | 数字 | 宽度异常 |
| `jsonConfig.defaultStyleData.base.height` | 数字 | 高度异常，控制点失效 |
| `jsonConfig.defaultStyleData.sub` | 对象（可空） | 可能报错 |
| `jsonConfig.defaultStyleData.pseudo` | 对象（可空） | 可能报错 |
| `jsonConfig.mappedProps` | 数组（可空） | 必须存在 |
| `jsonConfig.structuralCSS` | 对象（可空） | 必须存在 |
| `jsonConfig.computedProps` | 数组（可空） | 必须存在 |
| `jsonConfig.defaultProps.visible` | `true` | 控件不可见 |
| `jsonConfig.defaultProps.disabled` | `false` | 可能类型错误 |
| `htmlTemplate` | 字符串 | 控件空白 |

## 九、开发流程总结

1. **复制最小模板** → 改 `type`、`label`、`icon` → 测试能否拖入
2. **逐步增加属性**：
   - 加一个 `defaultProps` 业务字段 → 刷新看属性面板
   - 加 `htmlTemplate` 中的 `{{字段}}` → 看画布更新
   - 加 `mappedProps` 定义可编辑样式 → 在样式编辑器中测试
   - 加 `structuralCSS` 固定布局
   - 加 `cssString` 实现联动和动画
3. 每次修改后硬刷新浏览器（**Ctrl+F5**）
4. 如果报错，先注释新增代码，回到上一个可用状态
