## 一、项目概述

SUIDesigner 是一个基于 **Vue 3 + TypeScript + Pinia + Element Plus** 的 WYSIWYG（所见即所得）UI 设计器。用户可通过拖拽方式在画布上放置控件，实时编辑属性、样式和 CSS 源码，最终导出为独立 HTML 文件。

### 1.1 技术栈

| 技术 | 用途 |
|------|------|
| Vue 3 (^3.5.13) | 前端框架（Composition API + `<script setup>`） |
| TypeScript (~5.6.3) | 类型系统，编译时类型检查 |
| Pinia (^2.2.6) | 状态管理（widgetStore + themeStore） |
| Element Plus (^2.9.1) | UI 组件库（导航按钮、对话框、折叠面板） |
| Vite (^6.0.5) | 构建工具 + 开发服务器 |
| Monaco Editor (^0.55.1) | CSS 代码编辑器（语法高亮、CSS IntelliSense） |
| vue-tsc (^2.2.0) | TypeScript 类型检查 |
| vuedraggable (^4.1.0) | 拖拽排序 |

### 1.2 构建配置

- **构建命令**: `npm run build`（执行 `vue-tsc --noEmit && vite build`）
- **开发命令**: `npm run dev`（Vite 开发服务器，端口 5173）
- **模块解析**: `bundler` 模式，路径别名 `@/` → `src/`
- **JSON 导入**: `resolveJsonModule: true`（支持 JSON 配置文件直接导入）
- **Monaco Editor**: 通过 `vite-plugin-monaco-editor` 插件自动处理 worker 和语言服务

### 3.2 控件样式模型 — 双轨制

```
┌─────────────────────────────────────────────────────────┐
│  widget.style (WidgetStyle)     ← 旧格式（兼容层）       │
│  - 扁平对象：left/top/width/height/backgroundColor/...  │
│  - 含 subStyles / pseudoStyles 扩展字段                  │
│  - 控件组件通过 :style="widget.style" 直接绑定内联样式     │
│  - 与 widget.styleData.base 保持实时同步（双写机制）      │
├─────────────────────────────────────────────────────────┤
│  widget.styleData (WidgetStyleData) ← ★ 新格式（主数据源）│
│  - 结构化三层：{ base, sub, pseudo }                     │
│  - 纯 JSON 数据，不含 CSS 字符串                         │
│  - cssGenerator.ts 从这里生成 customCSS 字符串            │
│  - 目标：最终成为唯一数据源                                │
└─────────────────────────────────────────────────────────┘
```

**WidgetStyle**（旧格式，兼容层）:
```typescript
interface WidgetStyle {
  left: number; top: number; width: number; height: number
  backgroundColor?: string; color?: string; borderColor?: string
  borderWidth?: number; borderStyle?: string; borderRadius?: number

  textAlign?: string; textDecoration?: string

  minWidth?: number; minHeight?: number
  subStyles?: Record<string, Record<string, any>>    // 子选择器样式（如 '.progress-fill'）
  pseudoStyles?: Record<string, Record<string, any>>  // 伪类样式（无冒号key如 'hover'）
  [key: string]: any
}
```

**WidgetStyleData**（★ 新格式，纯JSON主数据源）:
```typescript
interface WidgetStyleData {
  /** 控件自身样式属性（camelCase key） */
  base: Record<string, any>
  /** 子选择器样式（key为选择器如 '.progress-fill'，value为属性对象） */
  sub: Record<string, Record<string, any>>
  /** 伪类样式（key为伪类如 ':hover'，value为属性对象） */
  pseudo: Record<string, Record<string, any>>
}
```

### 3.4 JSON 配置类型（★ 新架构核心）

```typescript
interface WidgetJSONConfig {
  label: string                        // 控件库显示名称
  icon: string                         // 控件库图标
  defaultStyleData: WidgetStyleData    // 默认样式数据（base + sub + pseudo）
  defaultStyleLight?: WidgetStyleData  // 浅色主题默认样式覆盖
  defaultStyleDark?: WidgetStyleData   // 深色主题默认样式覆盖
  structuralCSS: StructuralCSS         // 静态CSS规则（选择器 → CSS声明数组）
  mappedProps: MappedProp[]            // 动态映射属性（从styleData.base读取 → 对应CSS属性）
  computedProps: ComputedProp[]        // 计算属性（含模板变量如 {{halfH}}）
  defaultProps: Record<string, any>    // 默认业务属性（text, disabled等）
}

/** 结构CSS：不含变量，纯静态声明 */
type StructuralCSS = Record<string, string[]>
// 例：{ "#id": ["display: inline-flex", "cursor: pointer"], ".slider::before": ["content: ''", ...] }

/** 映射属性：从 styleData.base 读取动态值映射到CSS属性 */
interface MappedProp {
  selector: string     // CSS选择器（如 '#id', '.slider'）
  jsonKey: string      // styleData.base 中的key
  cssProp: string      // 对应的CSS属性名
  default: any         // 默认值
  unit?: string        // 数值单位（如 'px'）
}

/** 计算属性：含模板变量的CSS属性 */
interface ComputedProp {
  selector: string     // CSS选择器
  cssProp: string      // CSS属性名
  template: string     // 模板字符串（如 '{{halfH}}px', '{{headerHeight}}px'）
}
```

## 四、状态管理层（Pinia Stores）

### 4.1 widgetStore (`src/stores/widgetStore.ts`)

**核心职责**: 管理所有控件和画布的状态，负责控件的 CRUD、样式双向同步、历史记录、序列化/反序列化、性能优化。

#### 4.1.1 状态变量（★ 响应式数据）

| 变量名 | 类型 | 响应式 | 说明 |
|--------|------|:---:|------|
| `canvas` | `reactive<CanvasConfig>` | ✓ | ★ 画布配置（尺寸、背景、标题栏、透明度、毛玻璃等所有属性） |
| `widgets` | `reactive<Widget[]>` | ✓ | ★ 所有顶层控件数组（含嵌套 children） |
| `selectedIds` | `ref<string[]>` | ✓ | 当前选中控件ID列表（支持多选） |
| `history` | `ref<string[]>` | ✓ | 撤销/重做历史栈（JSON字符串数组，最多50条） |
| `historyIndex` | `ref<number>` | ✓ | 当前历史位置（-1=无历史） |
| `zIndexCounter` | `ref<number>` | ✓ | Z轴层级计数器（自增，不重复） |
| `tabHeaderHeights` | `reactive<Record<string,number>>` | ✓ | 标签页容器头部实际高度缓存（DOM测量值，widgetId→px） |
| `messageBoxConfig` | `reactive<MessageBoxConfig>` | ✓ | ★ 全局信息提示框配置 |
| `messageBoxPanelVisible` | `ref<boolean>` | ✓ | 是否显示信息框属性面板 |
| `inputBoxConfig` | `reactive<InputBoxConfig>` | ✓ | ★ 全局输入框配置 |
| `inputBoxPanelVisible` | `ref<boolean>` | ✓ | 是否显示输入框属性面板 |
| `contextMenuDefaultCSS` | `ref<string>` | ✓ | ★ 右键菜单默认弹出样式CSS（全局持久化） |
| `tooltipDefaultCSS` | `ref<string>` | ✓ | ★ 气泡框默认弹出样式CSS（全局持久化） |
| `_cssEditingWidgets` | `Set<string>` | ✗ | ★ 防循环触发集合（CSS编辑器↔属性面板，按控件ID隔离） |
| `_pendingStyleQueue` | `Record<string, Partial<WidgetStyle>>` | ✗ | ★ CSS编辑器编辑期间入队的样式变更，编辑结束后自动应用 |
| `_nodeIdCounter` | 模块变量 | ✗ | 树节点ID自增计数器 |

#### 4.1.3 核心方法一览

**控件创建**:
| 方法 | 说明 |
|------|------|
| `createDefaultWidget(type, x, y, themeMode?)` | ★ 控件创建核心工厂：从JSON配置初始化 styleData → 用 cssGenerator 生成 customCSS → 从 widgetsDefaultConfig 读取旧格式 style/defaultProps |
| `addWidget(type, x, y)` | 添加控件到画布顶层，返回创建的 Widget |
| `addWidgetAsChild(parentId, type, x, y, tabIndex)` | 添加子控件到标签页容器的指定标签页内 |
| `addWidgetToCardContainer(parentId, type, x, y)` | 添加子控件到卡片框内 |
| `addWidgets(newWidgets[])` | 批量添加控件（Ctrl+V 粘贴用） |

**控件更新（★ 样式双向同步 + 双速通道性能优化）**:
| 方法 | 说明 |
|------|------|
| `updateWidget(id, updates)` | 更新控件属性（style 合并，saveState） |
| `updateWidgetStyle(id, styleProps)` | ★ **双速通道架构**：立即层（rAF每帧1次DOM更新）→ 同步 widget.style + widget.styleData.base → 防抖层（150ms后CSS重建） |
| `updateWidgetCustomCSS(id, cssText)` | ★ **CSS编辑器→样式反向同步**：解析CSS → 写入 widget.style + widget.styleData（base/sub/pseudo）→ 取消挂起的rAF帧和防抖定时器 |
| `_regenerateCustomCSS(id)` | ★ 内部CSS重建：parseWidgetCSSEnhanced → mergeStyleToCSSEnhanced → 写回 customCSS |
| `setWidgetStyleProp(id, key, value)` | ★ **原子双写函数**：同时写入 widget.style 和 widget.styleData.base |

**控件删除**:
| 方法 | 说明 |
|------|------|
| `removeWidget(id)` | 递归删除控件（含所有后代子控件） |
| `removeWidgets(ids[])` | 批量删除 |

**选中管理**:

## 五、样式数据流全链路（★ 核心机制）

### 5.1 控件创建时的样式初始化

```
用户拖拽控件到画布（或Ctrl+V粘贴）
  │
  ▼
createDefaultWidget(type, x, y)                     [widgetStore.ts]
  │
  ├─ 从 widgetsDefaultConfig[type] 读取 label/icon/defaultStyle/defaultProps

  │   ├─ 'light' → config.defaultStyleLight
  │   └─ 其他   → undefined（使用 JSON 的 defaultStyleData）
  │
  ├─ 合并默认样式到 widget.style（旧格式，兼容层）
  │
  ├─ 从 mergeDefaults 读取 JSON 配置
  │   └─ getWidgetDefaultConfig(type) → WidgetJSONConfig
  │
  ├─ 初始化 styleData（★ 深拷贝 JSON 的 defaultStyleData）
  │   └─ widget.styleData = { base: {...}, sub: {...}, pseudo: {...} }
  │
  ├─ 同步初始位置/尺寸到 styleData.base
  │   └─ styleData.base.left = x; .top = y; .width = ...; .height = ...; .zIndex = ...
  │
  ├─ ★ 使用通用生成器生成 customCSS
  │   └─ cssGenerator.generateWidgetCSS(widget, jsonConfig) → CSS字符串
  │
  └─ widget.customCSS = 生成的CSS字符串
```

### 5.2 属性面板修改样式（★ 双速通道正向同步）

```
用户在属性面板修改样式（如 backgroundColor, fontSize 等）
  │
  ▼
StyleForm.onChange('backgroundColor', '#ff0000')
  │
  ▼
store.updateWidgetStyle(id, { backgroundColor: '#ff0000' })  [widgetStore.ts]
  │
  ├─ ★ _cssEditingWidgets.has(id) 检查 — 如该控件正在被CSS编辑器编辑，
  │   则样式变更入队 _pendingStyleQueue[id]（不会丢失），等待编辑结束后自动应用
  │
  ├─ ======== 立即层（rAF 限流，每帧最多1次 DOM 更新）========
  │   ├─ 合并帧内多次调用到 _styleFrameMap[id].props（Object.assign 累积）
  │   ├─ 同一帧内多次调用 → 取消上一帧 rAF → 重新注册
  │   └─ rAF 回调执行：
  │       ├─ Object.assign(widget.style, mergedProps)  ← 控件内联样式实时生效
  │       └─ ★ 同步到 widget.styleData.base（双写）
  │           for (const [key, value] of merged) {
  │             if (value !== undefined && key !== 'subStyles' && key !== 'pseudoStyles')
  │               widget.styleData.base[key] = value
  │           }
  │
  ├─ ======== 防抖层（150ms 后 CSS 重建）========
  │   ├─ 合并窗口内多次调用到 _styleDebounceMap[id].props（累积覆盖）
  │   ├─ 每次调用重置 150ms 定时器（clearTimeout + setTimeout）
  │   └─ 定时器到期：
  │       └─ _regenerateCustomCSS(id)
  │           ├─ parseWidgetCSSEnhanced(currentCSS) → { baseProps, subStyles, pseudoStyles }
  │           ├─ 合并 widget.style.subStyles / pseudoStyles
  │           └─ mergeStyleToCSSEnhanced() → 更新 widget.customCSS
  │
  ▼
CanvasArea.allWidgetCSS 计算属性自动检测 customCSS 变化
  │
  ▼
watch → 注入到 <style data-source="custom-css"> 标签
  │
  ▼
浏览器渲染 → 设计区实时呈现
```

### 5.3 CSS 编辑器修改（★ 反向同步）

```
用户在 CSS 编辑器修改源码（Monaco Editor）
  │
  ▼
CssEditor.vue 触发 @change 事件（300ms 防抖）
  │
  ▼
store.updateWidgetCustomCSS(id, cssText)              [widgetStore.ts]
  │
  ├─ 将控件ID加入 _cssEditingWidgets 集合（按控件隔离，防止循环触发）
  ├─ 清除该控件之前的挂起队列 delete _pendingStyleQueue[id]
  ├─ 取消挂起的 rAF 帧（_styleFrameMap.delete(id) + cancelAnimationFrame）
  ├─ 取消挂起的防抖定时器（_styleDebounceMap.delete(id) + clearTimeout）
  ├─ 保存 widget.customCSS = cssText
  │
  ├─ parseWidgetCSSEnhanced(cssText, widgetId, widget.type)
  │   └─ → { baseProps, subStyles, pseudoStyles }
  │
  ├─ ★ 写入 widget.style（baseProps：位置/尺寸/颜色等 kebab→camelCase 转换）
  ├─ 写入 widget.style.subStyles / pseudoStyles
  │
  ├─ ★ 同步到 widget.styleData.base（baseProps）
  ├─ ★ 同步到 widget.styleData.sub（subStyles，key含冒号如':hover'）
  ├─ ★ 同步到 widget.styleData.pseudo（pseudoStyles，key含冒号如':hover'）
  │
  └─ nextTick → 从 _cssEditingWidgets 移除该控件ID
      └─ 如果编辑期间有入队的样式变更（_pendingStyleQueue[id]），
         则自动应用（flush），确保用户在CSS编辑器编辑期间切换到属性
         面板修改的属性不丢失
  │
  ▼
CanvasArea 注入更新后的 CSS 到 DOM → 设计区实时渲染
```

### 5.4 拖拽/拉伸时的样式更新（rAF 限流）

## 六、性能优化架构（★ 新增）

### 6.1 双速通道架构

为解决高频操作（拖拽滑块、快速拖拽控件、颜色选择器快速点击、方向键连续微调）导致的页面卡死/崩溃问题，实现了**三层频率控制**：

```
┌──────────────────────────────────────────────────────────────┐
│  第1层：rAF 限流（立即视觉反馈）                                │
│  - 使用 requestAnimationFrame 限制每帧最多1次 DOM 更新          │
│  - 适用场景：属性面板样式修改、画布拖拽/拉伸、方向键微调         │
│  - 实现：_styleFrameMap（widgetStore）、_dragRAFId/_resizeRAFId（CanvasArea） │
│  - 效果：帧内多次调用合并为1次 Object.assign，浏览器从不掉帧     │
│  - 代码位置：widgetStore.ts（updateWidgetStyle）、CanvasArea.vue（onWidgetDragMove/onResizeMove）│
├──────────────────────────────────────────────────────────────┤
│  第2层：防抖 CSS 重建（延迟重计算）                             │
│  - 150ms 防抖窗口，窗口内连续变更累积覆盖                       │
│  - 窗口关闭后一次性执行 parseWidgetCSSEnhanced + mergeStyleToCSSEnhanced │
│  - 实现：_styleDebounceMap（widgetStore）                      │
│  - 效果：避免高频 CSS 解析/合并操作，大幅降低 CPU 开销          │
│  - 代码位置：widgetStore.ts（updateWidgetStyle）               │
├──────────────────────────────────────────────────────────────┤
│  第3层：rAF 合并 saveState（历史记录）                          │
│  - 帧内多次 saveState 调用合并为1次                            │
│  - 实现：_globalSaveRAFId（widgetStore）、scheduleSaveState（PropertyPanel）│
│  - 效果：避免高频 JSON 序列化（含递归序列化所有控件），减少内存占用│
│  - 代码位置：widgetStore.ts（saveState）、PropertyPanel.vue（scheduleSaveState）│
└──────────────────────────────────────────────────────────────┘
```

| 操作类型 | 优化方式 | 代码位置 | 说明 |
|----------|----------|----------|------|
| 属性面板样式修改 | rAF 限流 + 150ms防抖 | widgetStore.updateWidgetStyle | 双速通道：立即更新widget.style + 延迟重建CSS |

### 8.4 自定义控件序列化增强（v9.2）

**问题**：`serializeWidget` 使用硬编码属性列表，自定义控件的业务属性（如 `maxStars`、`starColor`、`showCount`）不在列表中，刷新后全部丢失。

**修复**：在序列化末尾收集 Widget 接口之外的额外属性，打包进 `_extraProps`，反序列化时自动恢复：

```typescript
// 序列化
const KNOWN_KEYS = new Set(Object.keys(data))
for (const key of Object.keys(w)) {
  if (!KNOWN_KEYS.has(key)) extraProps[key] = w[key]
}
if (Object.keys(extraProps).length > 0) data._extraProps = extraProps

// 反序列化
if (data._extraProps) {
  for (const [key, value] of Object.entries(data._extraProps)) {
    widget[key] = value
  }
}
```

### 8.2 cssGenerator.ts（★ 通用 CSS 生成器 — 新架构核心）

**主函数**: `generateWidgetCSS(widget: Widget, config: WidgetJSONConfig): string`

**CSS 选择器作用域**: 使用 `resolveSelector()` 函数自动为所有非 `#id` 选择器添加控件 ID 前缀，防止全局样式污染：
- `input` → `#toggle_1 input`
- `.slider` → `#toggle_1 .slider`
- `input:checked + .slider` → `#toggle_1 input:checked + .slider`
- `:hover` → `#button_1:hover`

**生成顺序**（严格按此顺序，确保 CSS 优先级正确）:
1. 定位声明块（`posDecl` — position/left/top/width/height/box-sizing）
2. 结构CSS（`structuralCSS['#id']` — 静态声明，不含变量，如 display/cursor）
3. 映射属性（`mappedProps` 中 selector 为 `'#id'` 的 — 从 `styleData.base` 动态取值）
4. 计算属性（`computedProps` 中 selector 为 `'#id'` 的 — 模板变量替换）
5. `styleData.base` 中未被 `mappedProps` 覆盖的额外属性（自动 fallback）
6. 非 `#id` 的子选择器结构CSS（通过 `resolveSelector` 添加作用域前缀）
7. 非 `#id` 的子选择器映射/计算属性（通过 `resolveSelector` 添加作用域前缀）
8. 伪类样式（`styleData.pseudo` — `:hover`, `:active`, `:disabled` 等）
9. 子选择器样式（`styleData.sub` — `.progress-fill`, `.card-header` 等）

| 辅助函数 | 说明 |
|----------|------|
| `posDecl(sd)` | 生成定位声明块（position:absolute + left/top/width/height/z-index） |
| `sv(sd, key, fallback?)` | 安全取值（从 styleData.base 取值，不存在则返回 fallback） |
| `resolveSelector(sel, id)` | ★ 选择器作用域：自动添加控件ID前缀，防止全局污染 |
| `resolveTemplateVars(widget, template)` | ★ 模板变量替换（`{{halfH}}`→`12px`, `{{headerHeight}}`→`35`, `{{barWidth}}`→`50%`等） |
| `renderMappedProp(widget, mp, sd)` | 渲染单个映射属性（从 `sd.base[mp.jsonKey]` 取值，附加 unit 后缀） |
| `renderComputedProp(widget, cp)` | 渲染单个计算属性（模板变量替换后输出 CSS 声明） |
| `buildRuleBlock(selector, declarations[])` | 构建完整 CSS 规则块（选择器 + 花括号 + 声明） |
| `validateSelectors(widget, config)` | ★ DEV 模式验证：检查 mappedProps/computedProps 中的 selector 是否在 structuralCSS 中定义 |

**额外导出函数**:
| 函数 | 说明 |
|------|------|

| `generateCanvasCSS(canvasCSS)` | 画布CSS透传（画布CSS涉及不同选择器模式，保留原有逻辑） |

**rAF 限流实现**（PropertyPanel.vue 内）:
| 函数 | 说明 |
|------|------|
| `scheduleCanvasChange(key, value, cssSync, extraCallback)` | mousedown → mousemove → mouseup 单个属性拖拽时调用的rAF合并 |
| `scheduleSaveState()` | 所有属性修改结束时调用，与 widgetStore._globalSaveRAFId 协作合并 |
| `onCanvasChange(key, event)` | 画布属性变更入口（checkbox/数值/颜色等），调用 scheduleCanvasChange |
| `onWBCssChange(cssText)` | 画布CSS编辑器变更 → scheduleCanvasChange → scheduleSaveState |
| `onMBCssChange(cssText)` | 信息框CSS变更 → widgetStore.messageBoxConfig.customCSS → scheduleSaveState |
| `onIBCssChange(cssText)` | 输入框CSS变更 → widgetStore.inputBoxConfig.customCSS → scheduleSaveState |

## 十一、历史记录与撤销/重做

### 11.1 机制

```
saveState()
  → 序列化完整设计：JSON.stringify({ canvas, widgets, messageBoxConfig, inputBoxConfig })
  → history.push(stateStr)
  → historyIndex = history.length - 1
  → triggerAutoSave()（防抖500ms后保存到localStorage）

undo()
  → historyIndex--
  → restoreState(history[historyIndex])
  → 完整恢复画布+控件+全局配置+主题快照

redo()
  → historyIndex++
  → restoreState(history[historyIndex])
```

### 11.2 触发时机

- 控件增删（addWidget/removeWidget/addWidgets）
- 控件属性修改（updateWidget → saveState）
- 样式修改（updateWidgetStyle 的防抖层回调 → saveState）
- CSS编辑器修改（updateWidgetCustomCSS → saveState）
- 拖拽/拉伸结束（CanvasArea.onWidgetDragEnd/onResizeEnd → saveState）
- 主题切换（themeStore.applyTheme → saveState）
- 粘贴控件（useKeyboard.pasteFromClipboard → saveState）
- 删除控件（useKeyboard.deleteSelected / 右键菜单删除 → saveState）
- 对齐/等宽高操作 → saveState

### 11.3 限制

- 最大历史记录数：**50 条**（`maxHistory = 50`）
- 超出时自动移除最早记录（`history.shift()`）
- 新操作会清除当前位置之后的所有历史（undo后做新操作→后面的redo记录被丢弃）
- saveState 受 rAF 合并：帧内多次调用合并为1次，避免高频序列化

## 十二、缓存与持久化

### 12.1 自动保存

```
用户操作
  → saveState()（保存历史，rAF合并：帧内多次调用合并为1次）
  → triggerAutoSave()（触发自动保存定时器）
  → 500ms 防抖（clearTimeout + setTimeout）
  → saveToLocalStorage()
      ├─ 计算数据指纹（JSON.stringify + 简单hash去重）
      ├─ 指纹相同则跳过写入（节省IO）
      └─ localStorage.setItem('ui-designer-cache', JSON.stringify(state))
```

### 12.2 页面加载恢复

```
App.vue onMounted()
  ├─ themeStore.initTheme()
  │   └─ 从 localStorage['ui-designer-theme'] 恢复主题
  │       └─ 非默认主题 → 延迟100ms调用 applyTheme()
  │
├─ loadGlobalConfigs()
│   └─ 从 localStorage['ui-designer-global-configs'] 恢复：
│       ├─ 信息框/输入框配置 → widgetStore.messageBoxConfig / inputBoxConfig
│       ├─ 画布外观属性（borderWidth/borderRadius/canvasFixedSize/showTitleBar/titleBarAlign）
│       ├─ 右键菜单默认弹出CSS → widgetStore.contextMenuDefaultCSS
│       └─ 气泡框默认弹出CSS → widgetStore.tooltipDefaultCSS
  │
  └─ 检查 localStorage['ui-designer-cache']
      ├─ 有缓存 + 非空数据 → 弹出"恢复设计"确认对话框（Element Plus MessageBox）
      ├─ 有缓存 + 空数据 + 有默认画布 → 静默初始化
      └─ 无缓存 → 初始化空画布（默认800x500）
```

### 12.3 页面关闭保存

```
window.addEventListener('beforeunload', () => {
  saveToLocalStorage(true)  // 强制保存，忽略去重指纹检查
})
```

## 十七、架构特点与维护注意事项

### 15.1 纯 JSON 配置方案的优势

1. **配置与代码分离**: 24 种控件的样式定义在独立的 JSON 文件中，修改默认视觉样式无需改 TypeScript 代码
2. **通用 CSS 生成器**: 一个 `cssGenerator.ts` 的 `generateWidgetCSS()` 函数替代了 24 个硬编码的 CSS 模板生成函数（减少约 1600 行冗余代码）
3. **类型安全**: JSON 配置文件通过 `WidgetJSONConfig` 接口约束，编译时校验结构正确性
4. **三向同步**: `widget.style` ↔ `widget.styleData` ↔ `widget.customCSS` 保持实时一致
5. **向后兼容**: 保留 `widget.style` 作为兼容层，控件组件零改动即可适配新架构
6. **主题友好**: `themes.ts` 的 `components` 字段使用 `WidgetStyleData` 结构，可直接覆盖 `styleData`
7. **CSS 选择器作用域**: `resolveSelector()` 自动为所有选择器添加控件 ID 前缀（如 `#toggle_1 .slider`），防止全局样式污染
8. **选择器一致性验证**: `validateSelectors()` 在 DEV 模式下自动检查 `mappedProps`/`computedProps` 中的选择器是否与 `structuralCSS` 定义一致

### 15.2 双速通道性能架构

```
rAF 限流（立即视觉反馈）
  └─ 每帧最多1次 DOM 更新
  └─ 帧内多次调用合并为1次 Object.assign
  └─ 适用：属性面板修改、画布拖拽/拉伸、方向键微调

防抖 CSS 重建（延迟重计算）
  └─ 150ms 防抖窗口
  └─ 窗口内变更累积覆盖（Object.assign 合并）
  └─ 窗口关闭后一次性执行 parseWidgetCSSEnhanced + mergeStyleToCSSEnhanced

rAF 合并 saveState
  └─ 帧内多次 saveState 合并为1次
  └─ 避免高频 JSON 序列化（含递归序列化所有控件）
```

### 15.3 关键维护注意事项

1. **双写同步的完整性**: 所有修改 `widget.style` 的地方必须同步修改 `widget.styleData.base`。已通过 `setWidgetStyleProp` 原子函数和 `updateWidgetStyle` rAF 回调中的批量双写实现。新增代码路径时优先使用 `setWidgetStyleProp`。

2. **两个 generateWidgetCSS 函数并存**:
   - `cssParser.ts` 中的版本：从旧 `widget.style`（含 subStyles/pseudoStyles）生成 CSS，仍被 `themeStore.applyThemeOverrides` 内部调用
   - `cssGenerator.ts` 中的版本：从新 `widget.styleData`（含 JSON 配置的结构CSS/映射/计算属性）生成 CSS，用于控件创建、导出和属性面板修改后的 CSS 重建
   - 未来可完全废弃旧版本，统一到新生成器

3. **伪类样式键名格式差异**:
   - `widget.style.pseudoStyles` 使用无冒号格式（如 `'hover'`）
   - `widget.styleData.pseudo` 使用有冒号格式（如 `':hover'`）
   - 在同步代码中需注意格式转换，避免遗漏导致样式不生效

4. **Monaco Editor 的循环依赖风险**:
   - `CssEditor.vue` 编辑 CSS → `updateWidgetCustomCSS` → 更新 `widget.customCSS`
   - `widget.customCSS` 是编辑器的绑定值（`modelValue`）
   - 已用 `isUpdatingFromProp`（CssEditor内部）+ `_cssEditing`（widgetStore全局）双重标志位防止循环
   - 需留意异步时序（setTimeout 0ms 延迟恢复标志位）

5. **JSON 配置文件中的选择器必须与 Vue 组件 DOM 结构匹配**:
   - 例如 `button.json` 中的 `structuralCSS['#id']` 生成的规则必须对应 `ButtonWidget.vue` 根元素的 id
   - 若将来修改控件 DOM 结构（增加 wrapper、重命名 class），需同步更新 JSON 配置中的选择器
   - 同时需确保 `htmlExporter.ts`/`ssrRenderer.ts` 生成的 HTML 类名也保持一致

6. **localStorage 容量限制**:
   - 浏览器限制约 5-10MB（不同浏览器不同）
   - 单个设计包含大量控件和复杂样式时，序列化后的 JSON 可能接近限制
   - 未来可考虑压缩存储（LZ-String）或支持 IndexedDB

7. **rAF 限流缓存清理**:
   - 在控件删除或 CSS 编辑器修改时，需要取消挂起的 rAF 帧和防抖定时器
   - `updateWidgetCustomCSS` 中已实现：`cancelAnimationFrame(frameId)` + `clearTimeout(timer)`
   - 新增代码路径产生挂起的 rAF/防抖时需同步清理

8. **主题切换 CSS 重建**:
   - `themeStore.applyThemeOverrides` 已改用 `cssGenerator.generateWidgetCSS()` 新生成器
   - 从 `widget.styleData` + JSON 配置生成 CSS，确保结构 CSS（如 toggle 的 `.slider::before`）正确包含在内
   - 旧版本使用 `cssParser.generateWidgetCSS()` 缺少结构 CSS，已修复

9. **颜色选择器事件选择**:
   - `StyleForm.vue` 中的颜色选择器使用 `@change` 事件（非 `@input`）
   - 颜色输入框（文本）使用 `@input` 事件（配合 rAF 限流，可实时预览）
   - 原因：ColorPicker 的 `@input` 会在颜色选择/透明度调节时每像素触发一次，导致卡顿

10. **_cssEditingWidgets 集合与队列机制**:
    - 旧 `_cssEditing` 布尔值已被替换为 `_cssEditingWidgets` 集合 + `_pendingStyleQueue` 队列
    - 按控件ID隔离编辑状态，编辑控件A时不影响控件B的属性修改
    - 编辑期间被丢弃的样式变更入队等待，`nextTick` 后自动应用
    - 使用 `nextTick` 替代旧 `setTimeout(50ms)`，时序更可靠

11. **全局持久化的分层设计**:
    - **主缓存**（`ui-designer-cache`）：保存完整设计快照（含全部控件数据+画布+配置）
    - **全局配置**（`ui-designer-global-configs`）：跨设计持久化，保存：
      - 信息框/输入框配置（全部属性）
      - 画布外观属性（borderWidth/borderRadius/canvasFixedSize/showTitleBar/titleBarAlign）
      - 右键菜单/气泡框默认弹出样式CSS
    - `isCanvasDefault()` 在检测是否显示"恢复设计"弹窗时，**排除**全局持久化的画布外观属性，只检测设计特有属性（`width`/`height`/`backgroundColor`/`borderColor`/`title`）

12. **右键菜单/气泡框实时预览弹窗**:
    - 删除原`ctxActiveTab`/`ttActiveTab`标签页系统和内联CssEditor
    - 点击"预览"打开模态弹窗（左侧实时预览 + 右侧CSS编辑器）
    - CSS变化通过 `updateCtxPreviewStyle()` / `updateTooltipPreviewStyle()` 实时更新预览
    - `CTX_PREVIEW_DEFAULT_CSS` / `TT_PREVIEW_DEFAULT_CSS` 常量存储预览默认样式
    - 弹窗标题栏支持拖拽（与 CssEditor 最大化拖拽使用相同模式）
    - 弹窗通过蒙版点击（@click.self）或关闭按钮关闭

13. **CssEditor 最大化拖拽**:
    - 最大化时通过 `left`/`top` 直接定位（替代旧 `left:50% + transform:translateX(-50%)`）
    - 标题栏 `mousedown` 启动拖拽，全局 `mousemove`/`mouseup` 控制
    - 带视口边界限制（不超出屏幕）
    - 拖拽结束后 200ms `ignoreBackdropClick` 防误触遮罩关闭
    - 最大化/还原时清除拖拽内联样式

14. **新增控件类型的完整步骤**:
    1. 创建 JSON 配置文件 `config/widgetDefaults/xxx.json`
    2. 注册到 `config/mergeDefaults.ts`（import + allWidgetDefaults 映射）
    3. 创建 Vue 组件 `components/widgets/XxxWidget.vue`
    4. 添加 WidgetType 联合类型到 `types/index.ts`
    5. 添加配置到 `config/widgetsDefaultConfig.ts`（label/icon/defaultStyle/defaultProps）
    6. 确保 `CHILD_SELECTOR_MAP`（cssParser.ts）中的选择器与 JSON 配置一致
    7. 确保 `htmlExporter.ts` 和 `ssrRenderer.ts` 中的 HTML 生成函数支持新控件
    8. 确保 `PreviewWidget.vue` 中的控件渲染逻辑支持新控件类型

## 十八、★ 控件渲染过程与预览导出流程（v9.0 新增）

### 16.1 整体渲染数据流

```
JSON 配置 (widgetDefaults/*.json)
    │
    ▼
widgetStore (Pinia 状态管理)
    │  widget.styleData.base     ← 用户通过属性面板修改的样式
    │  widget.styleData.sub      ← 子选择器样式（如 .header）
    │  widget.styleData.pseudo   ← 伪类样式（如 :hover）
    │  widget.styleData.structuralCSS  ← 基础结构CSS选择器
    │  widget.styleData.mappedProps    ← 映射属性（选择器→属性映射）
    │  widget.styleData.computedProps  ← 计算属性（含模板变量解析）
    │
    ▼
cssGenerator.ts  generateWidgetCSS()
    │  1. 解析 structuralCSS（选择器 + CSS声明）
    │  2. 处理 mappedProps（选择器映射 + 属性转换）
    │  3. 处理 styleData.sub（子选择器样式）
    │  4. 处理 computedProps（模板变量解析 + 属性覆盖）
    │  5. 处理 pseudo 伪类（:hover/:active/:focus 等）
    │  6. 合并 customCSS（用户自定义CSS）
    │  7. resolveSelector() 作用域选择器（添加 #id 前缀）
    │
    ▼
设计区渲染 (CanvasArea.vue)           预览/导出渲染
    │                                       │
    ▼                                       ▼
Widget 组件 (Vue SFC)              ssrExporter.ts (Vue SSR)
    │  props: widget                     │  @vue/server-renderer
    │  style: 内联样式                    │  renderToString()
    │  class: 动态类名                    │
    │                                       ▼
    ▼                              PreviewCanvas.vue
DOM 实时渲染                              │  <html> 完整结构
（响应式 + rAF 限流）                     │  <head> 样式 + webviewBridge.js
                                          │  <body> 控件列表
                                          │
                                          ▼
                                    PreviewWidget.vue
                                          │  遍历所有控件
                                          │  渲染每个控件（含标题栏）
                                          │  渲染子控件（嵌套容器）
                                          │
                                          ▼
                                    ssrRenderer.ts
                                          │  renderWidgetHTML() 生成HTML
                                          │  renderWidgetTitleBar() 标题栏
                                          │  renderWidgetChildren() 子控件
                                          │
                                          ▼
                                    完整 HTML 页面
                                          │
                                          ▼
                                    htmlExporter.ts
                                          │  exportFullHTML() 最终导出
                                          │  downloadHTML() 下载文件
```

### 16.2 设计区渲染过程（CanvasArea.vue）

设计区使用 Vue 响应式渲染，控件以 Vue SFC 组件形式直接挂载到 DOM：

1. **控件树遍历**: `CanvasArea.vue` 遍历 `widgetStore.widgets` 数组，为每个控件渲染对应的 Vue 组件
2. **组件选择**: 通过 `widget.type` 动态选择组件（如 `ButtonWidget.vue`、`InputWidget.vue` 等）
3. **样式注入**: 通过 CSS 注入 `<style>` 标签（`injectCSS()`），将 `generateWidgetCSS()` 生成的样式注入到画布 DOM
4. **内联样式**: 每个控件组件接收 `widget.style` 作为 props，通过 `:style` 绑定应用到根元素
5. **rAF 限流**: 高频操作（拖拽、属性修改）通过 rAF 限流，每帧最多 1 次 DOM 更新
6. **嵌套容器**: `TabsContainer.vue` 和 `CardBoxWidget.vue` 支持递归渲染子控件

### 16.3 预览/导出渲染过程（Vue SSR 引擎）

v9.0 版本将预览和导出统一为 Vue SSR 渲染引擎，保证预览与导出 HTML 完全一致：

#### 步骤 1: ssrExporter.ts - Vue SSR 渲染引擎

```typescript
// 使用 @vue/server-renderer 将 Vue 组件渲染为 HTML 字符串
import { renderToString } from '@vue/server-renderer'
import { createSSRApp } from 'vue'

// 创建 SSR 应用实例
const app = createSSRApp(PreviewCanvas, {
  widgets: widgetStore.widgets,
  canvasConfig: widgetStore.canvasConfig,
  // ... 其他 props
})

// 渲染为 HTML 字符串
const html = await renderToString(app)
```

**`ssrExporter.ts` 核心职责**:
- 创建 Vue SSR 应用实例，挂载 `PreviewCanvas.vue`
- 调用 `@vue/server-renderer` 的 `renderToString()` 生成完整 HTML
- 收集所有 CSS 样式（`collectAllCSS()`）
- 检测运行时依赖（`detectRuntimeDependencies()`）
- 嵌入 `webviewBridge.js` 运行时脚本
- 处理 `messageBox`/`inputBox` 全局配置控件

#### 步骤 2: PreviewCanvas.vue - 预览画布组件

**`PreviewCanvas.vue` 核心职责**:
- 控制完整 HTML 页面结构（`<html>` → `<head>` + `<body>`）
- `<head>` 区域：注入全局样式、控件 CSS、`webviewBridge.js` 脚本
- `<body>` 区域：渲染所有控件（通过 `PreviewWidget.vue`）
- 处理画布配置（`canvasConfig`）：标题栏、背景色、窗口大小等
- 处理全局组件配置（`messageBox`/`inputBox` 的 HTML + CSS）

#### 步骤 3: PreviewWidget.vue - 预览控件组件

**`PreviewWidget.vue` 核心职责**:
- 遍历 `widgets` 数组，为每个控件调用 `ssrRenderer.ts` 生成 HTML
- 渲染控件标题栏（`renderWidgetTitleBar()`）
- 递归渲染嵌套容器的子控件
- 处理特殊控件类型（`contextMenu`/`tooltip` 仅生成占位标记）
- 为每个控件设置 `data-ctrl-type` 和 `data-ctrl-name` 属性（供 `webviewBridge.js` 运行时识别）

#### 步骤 4: ssrRenderer.ts - SSR 统一渲染引擎

**`ssrRenderer.ts` 核心职责**:
- `renderWidgetHTML(widget)`: 根据控件类型生成对应的 HTML 片段
- `renderWidgetTitleBar(widget)`: 渲染标题栏（卡片框、标签页容器等）
- `renderWidgetChildren(widget)`: 递归渲染嵌套子控件
- 处理控件类型特定的样式属性（如卡片框的 `headerColor`、`headerTitleColor`）
- 输出 HTML 包含 `data-ctrl-type`、`data-ctrl-name`、`data-ctrl-id` 等标记属性

#### 步骤 5: htmlExporter.ts - 最终导出

**`htmlExporter.ts` 核心职责**:
- `exportFullHTML()`: 调用 `ssrExporter.ts` 生成完整 HTML，添加下载逻辑
- `downloadHTML()`: 触发浏览器下载 .html 文件
- 处理 `webviewBridge.js` 运行时脚本的嵌入
- 检测并嵌入运行时依赖（`ui-runtime.ts` 中的 TreeManager/DataTableManager）

### 16.4 预览与导出的一致性保证

```
                    设计区渲染             预览             导出
                    ──────────           ────            ────
渲染引擎            Vue 响应式            Vue SSR         Vue SSR
组件                Widget.vue            PreviewWidget   PreviewWidget
CSS生成             cssGenerator.ts       cssGenerator    cssGenerator
CSS注入             injectCSS()           <style>内联      <style>内联
webviewBridge       ✗ 不需要              ✓ 嵌入          ✓ 嵌入
```

**关键保证**:
1. 预览和导出的 HTML 结构完全一致（都通过 `PreviewWidget.vue` + `ssrRenderer.ts` 生成）
2. 预览和导出的 CSS 完全一致（都通过 `cssGenerator.ts` 生成）
3. 预览和导出都嵌入 `webviewBridge.js`（提供运行时 API 和交互能力）
4. 设计区使用 Vue 响应式组件，预览/导出使用 SSR 静态 HTML，但控件 HTML 结构保持一致
