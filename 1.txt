项目简介:
本项目是个UI设计器，设计完成后预览生成会调用独立设计器之外的webviewBridge.js文件
`public/webviewBridge.js`:本运行时独立于 UI 设计器，负责监控所有控件事件并与 WebView2 宿主通信。请了解项目后开始对webviewBridge.js进行bug的修复

# UI 设计器项目总结（2026-05-23）

## 一、已完成的核心工作

### 1. 统一配置重构
- 建立了 `widgetsDefaultConfig.ts` 统一配置文件，集中管理所有控件的默认属性、样式和 CSS 模板。
- `createDefaultWidget` 已改为优先从配置读取，保留原有 switch 作为 fallback。
- 左侧控件库已改为从配置动态生成，消除硬编码。

### 2. 静默控件开发
完成了 5 个静默控件的实现，设计区可配置，运行时通过 API 调用：
- **文件选择器**：支持单选/多选、文件类型过滤，可选择将文件信息发送给宿主或填入指定控件。
- **气泡提示框**：支持悬停/点击/聚焦触发，位置可配置（上/下/左/右/自动），支持延迟显示/隐藏。
- **右键菜单**：支持静态菜单（设计区配置）和动态菜单（通过宿主通讯），支持多层级子菜单。
- **信息框**：全局唯一模态对话框，支持多种图标和按钮组合，支持宿主提供 `requestId` 进行异步回调匹配。
- **输入框**：基于信息框架构，支持文本/数字/密码输入，支持自定义提示文本和默认值。

### 3. 运行时框架完善
- `webviewBridge.js` 实现了统一包装层，自动处理图标占位符转换和返回值字符串化。
- 支持「便捷 API + 命令分发」双轨并行模式，兼容易语言、Python、C 调用。
- `send` 事件发送前自动将 emoji 转回占位符，确保宿主收到原始数据。
- 为信息框和输入框添加了样式编辑器的自动保存和“恢复默认样式”功能。

## 二、建议添加的新功能

### 高优先级（强烈推荐）

#### 1. 静默控件使用文档与示例
目前静默控件功能强大但用户可能不知道如何调用。建议在项目中增加一个“控件演示页面”，展示每个静默控件的配置和调用示例（包括 `show` 方法、`overrides` 参数、`requestId` 用法）。

#### 2. 设计区所见即所得增强
当前用户编写的 `customCSS` 中的伪类（如 `:hover`）在设计区不生效，只能导出预览才能看到效果。建议在画布中为每个控件动态注入其 `customCSS` 样式（通过 `<style>` 标签），实现真正的所见即所得。

#### 3. 模板/主题功能
支持导入/导出设计配置（JSON），用户可一键应用预设模板（如“深色主题”、“移动端布局”）。这能极大提高设计复用效率。

### 中优先级（视时间而定）

#### 4. 拖拽排序与图层管理面板
增加图层树（Outline）面板，树形结构显示所有控件（含容器内子控件），支持拖拽调整顺序、重命名、锁定、隐藏等。在复杂设计中，图层管理能显著提升工作效率。

#### 5. 响应式布局与相对定位
当前所有控件都是绝对定位，画布尺寸变化时控件不会自适应。建议增加“布局容器”控件（如 `FlexRow`、`Grid`），内部子控件使用相对定位，画布尺寸变化时自动调整。

#### 6. 多格式导出
当前只能导出独立 HTML，建议增加多种导出格式：纯 HTML、Vue SFC、React JSX、微信小程序 wxml。可先实现 Vue SFC，因为项目本身是 Vue 3。

### 低优先级（锦上添花）

#### 7. AI 辅助功能
在属性面板增加“AI 优化”按钮，将当前控件的样式或 `customCSS` 发送给 AI（如 OpenAI API），返回改进后的代码。

#### 8. 协作与版本管理
实现简单的“保存版本”功能，支持多人同时编辑同一设计（类似 Figma，工程量大）。

#### 9. 插件化架构
将 `widgetsDefaultConfig` 设计为可动态注册，允许第三方开发者开发自定义控件。

## 三、代码质量建议
- **单元测试**：为核心模块（`widgetStore`、`cssParser`、`htmlExporter`）编写单元测试。
- **性能优化**：对于大设计（100+ 控件），可考虑使用虚拟滚动。
- **文档**：完善项目 README，包括架构说明、控件扩展指南、API 参考。
